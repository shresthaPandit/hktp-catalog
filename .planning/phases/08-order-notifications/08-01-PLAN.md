    - On AbortError: error string is "Timeout after 5s"

    - Supabase client is created fresh inside this function (not passed in) via await createClient()
    - Top-level try/catch wraps the entire function body — any unexpected throw writes to whatsapp_error

After line 62 (after `await supabase.from('cart_items').delete().eq('user_id', user.id)`) and before `revalidatePath('/cart')`, build the payload and register the after() callback:

```typescript
// Build webhook payload
const notificationPayload: OrderNotificationPayload = {
  order_number: order.order_number,
  company_name: customerInfo.company_name,
  phone: customerInfo.phone,
  items: items.map(i => ({
    name: i.name,
    quantity: i.quantity,
    price_at_order: i.price_at_order,
    subtotal: i.price_at_order != null ? i.price_at_order * i.quantity : null,
  })),
  subtotal: items.reduce((sum, i) => sum + (i.price_at_order ?? 0) * i.quantity, 0),
  customer_notes: notes,
  placed_at: new Date().toISOString(),
}

// Fire-and-forget via after() — runs after redirect, keeps Vercel function alive
after(async () => {
  await sendOrderNotification(order.id, order.order_number, notificationPayload)
})
```

CRITICAL: after() is called BEFORE redirect() — that is the correct order. after() must be called synchronously in the server action body, not inside a nested async callback.
Do NOT await the sendOrderNotification call directly in createOrder.
Do NOT use void fetch(...) — use after() exclusively.
